Troubleshooting payonline

The EPFL payonline procedure works the following way :

  1. Enter your personal data on the registration web page and submit the form
  2. You get to payonline.epfl.ch which confirms that your data has been registered; click Continue to
  3. pop PostFinance's window : choose the credit card, then give the card's parameters
    (if your browser blocks the popups, you'll get a message which asks if you want to "Pay" or "Cancel")
  4. PostFinance validates the transaction and returns a result code to payonline.epfl.ch :
    • the card is valid, result code=1 (Transaction accepted),
    • the card is rejected or you cancel the operation, result code=0 (Transaction failed);
    • you close the window during the process, result code is empty (Transaction failed).

In case of payment troubles, contact the following, according to the failure point :

  • at the conference's site : the person in charge of the instance
  • at payonline.epfl.ch (EPFL logo) : 1234@epfl.ch
  • payment choice and credit card details (on PostFinances's site) : shopperhelp@postfinance.ch
Please provide transaction details (number, date, etc.), browser and operating system, a description of the problem and the error messages. A screen copy might be useful : an image is better than 1000 words.

Notes

  • JavaScript is required in order to use PostFinance's services (step 3)
  • Popup blocking might cause problems : please disable it
  • Please note that a credit card electronic transaction may be refused by the card issuer at step 4 (error codes between 200 and 299). In this case, you need to contact your card issuer for details.
  • In case of several transaction failures, please use one alternative payment method so you shouldn't miss the conference.
    Refer to the registration page for details.
