Problèmes avec payonline

Les étapes de la procédure de EPFL payonline :

  1. Saisir les données personnelles sur la page d'enregistrement et soumettre le formulaire
  2. Suit la page de payonline.epfl.ch qui confirme l'enregistrement des données; cliquer Continuer pour faire apparaitre
  3. la fenêtre pop-up de PostFinance : choisir la manière de paiement et donner ensuite les paramètres de la carte
    (si votre navigateur bloque les fenêtres pop-up le message pour "Payer" ou "Annuler" est affiché)
  4. PostFinance valide la transaction et retourne un code à payonline.epfl.ch :
    • la carte est solvable, code=1 (Transaction acceptée),
    • la carte n'est pas solvable ou la procédure est interrompue par "Annuler", code=0 (Transaction échouée);
    • vous interrompez la procédure en fermant la fenêtre de PostFinance, code vide (Transaction échouée).

En cas d'échec de paiement, veuillez contacter les responsables ci-dessous, selon l'étape où le problème est arrivé :

  • sur le site de la conférence : le responsable de l'instance de paiement
  • sur payonline.epfl.ch (EPFL logo) : 1234@epfl.ch
  • lors du choix du mode de paiement et/ou la saisie des détails de la carte de crédit (sur le site de PostFinance) : shopperhelp@postfinance.ch
Veuillez fournir des détails de la transaction (numéro, date, etc.), le navigateur et système informatique utilisés, une description du problème et la copie du message d'erreur. Une copie d'écran attachée à votre message remplace les quelques 1000 mots de trop.

Notes

  • JavaScript est exigé par PostFinance pour son service de paiement (étape 3)
  • Le blocage des fenêtres popup par le navigateur peut poser un problème : veuillez le désactiver, svp.
  • Veuillez prendre note que certaines cartes de crédit, selon l'organisme émetteur, peuvent être refusées pour les transactions électroniques lors de l'étape 4 (codes d'erreur entre 200 et 299). Dans ce cas il faut contacter l'organisme émetteur de la carte pour plus de détails.
  • En cas d'échecs répétés, utiliser un moyen alternatif de paiement pour ne pas rater l'événement.
    Ces détails devraient se trouver sur la page d'enregistrement.
